# Adobe Experience Manager (AEM) 분석서

**문서명**: Agentic CMS 벤치마크 요구사항 정의서  
**대상 솔루션**: Adobe Experience Manager Sites + Assets + Adobe Sensei/GenAI  
**버전**: v1.0  
**작성일**: 2026년 4월 29일  
**작성자**: ACMS 기획/개발팀

## 1. 문서 개요

### 1.1 목적
본 문서는 AEM의 핵심 기능을 Agentic CMS 설계 관점에서 분석하여, ACMS의 기능 우선순위와 구현 전략 수립에 활용하는 것을 목적으로 한다.

### 1.2 범위
- 콘텐츠 모델링/저작/워크플로우
- 멀티채널 전달(웹, 앱, API)
- AI 보조 작성, 태깅, 개인화
- 엔터프라이즈 거버넌스 및 운영

### 1.3 대상 독자
- 제품기획자, 솔루션 아키텍트, 프론트/백엔드 개발자, QA

## 2. 시스템 개요

### 2.1 시스템 목적
AEM은 대규모 기업의 브랜드 콘텐츠를 중앙 관리하고, 다채널로 일관되게 배포하며, AI 기반 자동화를 통해 제작 속도를 높이는 엔터프라이즈 CMS 플랫폼이다.

### 2.2 주요 이해관계자
- 콘텐츠 에디터: 빠른 작성/승인/배포
- 마케팅 팀: 캠페인/개인화/성과 개선
- 개발팀: API 기반 프론트엔드 통합
- 운영/보안팀: 권한/감사/규정 준수

### 2.3 성공 기준 (ACMS 관점)
- 콘텐츠 생성 리드타임 30% 이상 단축
- 재사용 가능한 콘텐츠 블록 비율 60% 이상
- 배포 오류율 1% 미만
- AI 제안 수용률 40% 이상

## 3. 기능적 요구사항 (벤치마크 추출)

### 3.1 콘텐츠 모델/저작

**REQ-AEM-CONTENT-001: 구조화 콘텐츠 모델**
- **설명**: Headless 전달을 위한 필드 기반 콘텐츠 모델 정의 지원
- **우선순위**: P0
- **상세 요구사항**:
  - 콘텐츠 타입/필드 스키마 버전 관리
  - 참조형 필드(연관 문서/자산) 지원
  - 지역/브랜드 단위 모델 확장
- **승인 조건**:
  - [ ] 모델 변경 시 하위 콘텐츠 영향도 확인 가능
  - [ ] 모델 버전별 API 응답 검증 가능

**REQ-AEM-CONTENT-002: 컴포넌트 기반 페이지 저작**
- **설명**: 페이지 편집기에서 재사용 가능한 컴포넌트 조립
- **우선순위**: P0
- **상세 요구사항**:
  - 드래그앤드롭 레이아웃
  - 템플릿/디자인 시스템 연계
  - 프리뷰 및 게시 전 검수
- **승인 조건**:
  - [ ] 비개발자도 표준 페이지 10분 이내 구성
  - [ ] 컴포넌트 정책 위반 시 게시 차단

### 3.2 자산 관리(DAM)

**REQ-AEM-ASSET-001: AI 기반 태깅/메타데이터 보강**
- **설명**: 이미지/문서 자산에 자동 태그와 설명 생성
- **우선순위**: P1
- **상세 요구사항**:
  - 객체/장면 인식 기반 자동 태깅
  - 메타데이터 규칙 기반 정규화
  - 부적절 콘텐츠 탐지 플래그
- **승인 조건**:
  - [ ] 자동 태그 정확도 목표치(예: 80%+) 측정 리포트 제공
  - [ ] 승인 전 태그 수정 이력 감사 가능

### 3.3 워크플로우/거버넌스

**REQ-AEM-WF-001: 다단계 승인 워크플로우**
- **설명**: 작성-검토-법무-배포의 단계별 승인 플로우 구성
- **우선순위**: P0
- **상세 요구사항**:
  - 역할별 승인 권한
  - SLA 기반 지연 알림
  - 반려 사유 기록 및 재승인
- **승인 조건**:
  - [ ] 승인 단계별 처리시간 추적 가능
  - [ ] 감사 로그(누가/언제/무엇) 조회 가능

### 3.4 멀티채널/전달

**REQ-AEM-DELIVERY-001: Headless API 제공**
- **설명**: GraphQL/REST로 콘텐츠를 다양한 프론트엔드에 전달
- **우선순위**: P0
- **상세 요구사항**:
  - 캐시 친화적 쿼리 지원
  - 채널별 변형 콘텐츠(웹/앱/키오스크)
  - 게시 시 CDN 무효화 자동화
- **승인 조건**:
  - [ ] API 평균 응답시간 목표치 충족
  - [ ] 게시 후 채널 반영 지연 1분 이내

### 3.5 AI/개인화

**REQ-AEM-AI-001: 생성형 콘텐츠 초안 지원**
- **설명**: 브랜드 가이드에 맞춘 제목/본문/요약 자동 제안
- **우선순위**: P1
- **상세 요구사항**:
  - 프롬프트 템플릿 관리
  - 금칙어/컴플라이언스 검사
  - 사람이 최종 승인하는 HITL(Human-in-the-loop)
- **승인 조건**:
  - [ ] AI 초안 사용 여부와 수정률 추적 가능
  - [ ] 정책 위반 텍스트 자동 차단

## 4. 비기능적 요구사항

- **성능**: 대용량 페이지/자산에서도 편집기 응답성 유지
- **보안**: SSO, RBAC, 감사 로그, 지역별 데이터 규정 대응
- **확장성**: 다국어/다브랜드/고트래픽 캠페인 확장
- **운영성**: 배포 파이프라인 자동화 및 롤백 전략 제공

## 5. ACMS 설계 시사점

- 엔터프라이즈급 워크플로우/권한/감사를 초기 아키텍처에 포함해야 함
- AI 기능은 "자동 생성"보다 "정책 준수 + 검수 가능한 제안"이 핵심
- 모델 기반 콘텐츠 설계를 먼저 확정하고, 페이지 저작은 상위 추상화로 제공
- 멀티채널 API 전달과 캐시 전략을 기능 요구와 함께 동시 설계 필요
