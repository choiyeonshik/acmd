# Optimizely CMS 분석서

**문서명**: Agentic CMS 벤치마크 요구사항 정의서  
**대상 솔루션**: Optimizely CMS + Experimentation + CMP  
**버전**: v1.0  
**작성일**: 2026년 4월 29일  
**작성자**: ACMS 기획/개발팀

## 1. 문서 개요

### 1.1 목적
Optimizely의 콘텐츠-실험-성과 측정 통합 구조를 분석하여 ACMS의 실험 중심 운영 모델을 설계한다.

### 1.2 범위
- 콘텐츠 관리 및 협업 플로우
- 실험/최적화 기능
- 콘텐츠 마케팅 계획(CMP) 연계
- 데이터 기반 의사결정 자동화

### 1.3 대상 독자
- 마케팅 운영팀, 제품분석팀, 플랫폼 개발팀

## 2. 시스템 개요

### 2.1 시스템 목적
Optimizely는 콘텐츠 제작부터 실험, 성과 분석까지 연결해 디지털 경험을 지속 최적화하는 DXP 성격의 CMS 솔루션이다.

### 2.2 주요 이해관계자
- 에디터/마케터: 캠페인 콘텐츠 제작 및 배포
- 데이터 분석가: 실험 성과 해석
- 개발자: 실험 대상 컴포넌트/이벤트 구현

### 2.3 성공 기준 (ACMS 관점)
- 실험 적용 페이지 비율 30% 이상
- 콘텐츠 성과 리뷰 주기 2주 이내 정착
- KPI 기반 자동 추천 적용률 25% 이상

## 3. 기능적 요구사항 (벤치마크 추출)

### 3.1 콘텐츠 협업

**REQ-OPT-COLLAB-001: 캠페인 중심 콘텐츠 캘린더**
- **설명**: 콘텐츠를 캠페인/채널/일정 단위로 계획/추적
- **우선순위**: P1
- **상세 요구사항**:
  - 콘텐츠 상태(아이디어/작성/검수/게시) 추적
  - 담당자 할당 및 SLA 관리
  - 일정 충돌 알림
- **승인 조건**:
  - [ ] 상태 전환 이력이 모두 기록됨
  - [ ] SLA 초과 작업 자동 알림 발송

### 3.2 실험 통합

**REQ-OPT-EXP-001: 콘텐츠 변형 실험**
- **설명**: 제목/배너/CTA 등 변형 실험 지원
- **우선순위**: P0
- **상세 요구사항**:
  - 실험 목표 및 세그먼트 설정
  - 변형별 트래픽 배분
  - 성과 기반 자동 승자 선택
- **승인 조건**:
  - [ ] 실험 전후 KPI 변화 비교 가능
  - [ ] 승자 반영 시 롤백 가능

### 3.3 추천/자동화

**REQ-OPT-AUTO-001: 성과 기반 콘텐츠 추천**
- **설명**: 과거 성과 지표를 기반으로 다음 액션 제안
- **우선순위**: P1
- **상세 요구사항**:
  - 저성과 콘텐츠 개선 제안
  - 채널별 최적 발행 시간 추천
  - AI 제안 근거(지표/데이터) 표시
- **승인 조건**:
  - [ ] 추천 수용 후 성과 개선 추적 가능
  - [ ] 추천 근거가 대시보드에서 확인 가능

### 3.4 엔터프라이즈 운영

**REQ-OPT-OPS-001: 다팀/다브랜드 운영 지원**
- **설명**: 조직별 권한 분리와 공통 자산 재사용
- **우선순위**: P0
- **상세 요구사항**:
  - 워크스페이스 단위 접근 제어
  - 브랜드별 템플릿 분리
  - 공통 컴포넌트 라이브러리 관리
- **승인 조건**:
  - [ ] 조직 간 무단 접근 차단
  - [ ] 공통 컴포넌트 변경 영향도 추적

## 4. 비기능적 요구사항

- **분석 정확성**: 이벤트 수집 누락률 최소화
- **성능**: 실험 스크립트 적용 시 렌더링 지연 최소화
- **운영성**: 캠페인 일정, 실험, 게시 로그 통합 조회
- **보안**: 역할 기반 민감 지표 접근 통제

## 5. ACMS 설계 시사점

- Agentic CMS에는 실험/최적화 루프가 기본 내장되어야 함
- AI 추천은 반드시 "근거 데이터 + 검수 가능성"을 같이 제공해야 신뢰 확보
- 콘텐츠 생산성뿐 아니라 성과지표 연결까지 포함한 운영 모델이 필요
- 캠페인/콘텐츠/실험 단위 엔터티를 분리 설계하면 자동화 확장에 유리
