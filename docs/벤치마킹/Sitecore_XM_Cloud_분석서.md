# Sitecore XM Cloud 분석서

**문서명**: Agentic CMS 벤치마크 요구사항 정의서  
**대상 솔루션**: Sitecore XM Cloud + Personalize + CDP  
**버전**: v1.0  
**작성일**: 2026년 4월 29일  
**작성자**: ACMS 기획/개발팀

## 1. 문서 개요

### 1.1 목적
Sitecore XM Cloud의 엔터프라이즈 웹 경험 관리와 개인화 기능을 분석해 ACMS의 경험 오케스트레이션 구조를 정의한다.

### 1.2 범위
- 헤드리스 페이지 빌딩
- 개인화/실험(A/B Test)
- 고객 데이터 연계(CDP)
- 조직 단위 거버넌스 및 운영

### 1.3 대상 독자
- CX 기획자, 솔루션 아키텍트, 데이터/마케팅 개발자

## 2. 시스템 개요

### 2.1 시스템 목적
Sitecore XM Cloud는 클라우드 네이티브 CMS와 개인화 기능을 결합해, 고객 맥락 기반의 디지털 경험을 제공하는 플랫폼이다.

### 2.2 주요 이해관계자
- 마케터: 세그먼트별 경험 최적화
- 에디터: 시각적 페이지 편집 및 게시
- 개발자: 컴포넌트/데이터 통합 구현
- 데이터팀: 행동 데이터 기반 의사결정

### 2.3 성공 기준 (ACMS 관점)
- 개인화 캠페인 전환율 15% 향상
- 실험 설정-배포 주기 40% 단축
- 경험 컴포넌트 재사용률 50% 이상

## 3. 기능적 요구사항 (벤치마크 추출)

### 3.1 경험 조립/저작

**REQ-SC-EXP-001: 비주얼 경험 편집**
- **설명**: 컴포넌트 단위로 페이지 경험을 구성하고 프리뷰
- **우선순위**: P0
- **상세 요구사항**:
  - 페이지/섹션/컴포넌트 계층 편집
  - 디바이스별 프리뷰
  - 게시 전 링크/콘텐츠 유효성 검사
- **승인 조건**:
  - [ ] 핵심 랜딩 페이지를 코드 수정 없이 편집 가능
  - [ ] 게시 전 오류 탐지율 목표치 충족

### 3.2 개인화/실험

**REQ-SC-PERSONA-001: 규칙 기반 개인화**
- **설명**: 세그먼트/컨텍스트에 따라 콘텐츠 변형 노출
- **우선순위**: P0
- **상세 요구사항**:
  - 방문자 속성/행동 기반 규칙
  - 우선순위 충돌 해결 정책
  - 노출 이력 기록
- **승인 조건**:
  - [ ] 동일 요청에서 개인화 판단 지연 최소화
  - [ ] 규칙 충돌 시 예측 가능한 결과 보장

**REQ-SC-AB-001: A/B/N 실험 지원**
- **설명**: 경험 변형 실험 및 성과 비교
- **우선순위**: P1
- **상세 요구사항**:
  - 트래픽 분배 비율 설정
  - 목표 지표(CTR, CVR 등) 설정
  - 통계적 유의성 체크
- **승인 조건**:
  - [ ] 실험 결과 대시보드에서 지표 확인 가능
  - [ ] 실험 승자 자동 적용 옵션 제공

### 3.3 데이터 연계

**REQ-SC-DATA-001: CDP 연동 기반 세그먼트 활용**
- **설명**: 고객 데이터 플랫폼과 연계해 정교한 타게팅
- **우선순위**: P1
- **상세 요구사항**:
  - 세그먼트 동기화 주기 관리
  - 개인정보 마스킹/동의 상태 반영
  - 캠페인 성과 피드백 루프
- **승인 조건**:
  - [ ] 동의 상태 변경 시 개인화 즉시 반영
  - [ ] 세그먼트 동기화 실패 알림 제공

### 3.4 운영/거버넌스

**REQ-SC-GOV-001: 엔터프라이즈 권한 체계**
- **설명**: 브랜드/지역/역할 단위 접근 및 승인 통제
- **우선순위**: P0
- **상세 요구사항**:
  - SSO 연동
  - 역할 기반 메뉴/기능 제한
  - 감사 로그 및 변경 이력 추적
- **승인 조건**:
  - [ ] 무권한 접근 차단 검증
  - [ ] 모든 게시 변경 이력 조회 가능

## 4. 비기능적 요구사항

- **성능**: 개인화 적용 시에도 페이지 로드 SLA 충족
- **신뢰성**: 외부 데이터 지연 시 폴백 경험 제공
- **보안/컴플라이언스**: 개인정보 처리 정책 내장
- **운영성**: 캠페인 실패 시 즉시 중지/롤백 가능

## 5. 외부 시스템 연동 요구사항

### 5.1 연동 대상 시스템

- IAM/SSO 및 감사 시스템

### 5.2 연동 요구사항
**REQ-SC-INT-001: 개인화 데이터 실시간 연동**
- **설명**: 세그먼트 및 동의 상태를 개인화 엔진에 반영
- **우선순위**: P0
- **상세 요구사항**:
  - 이벤트 지연 허용치 정의
  - 데이터 동기화 실패 감지/복구
  - 동의 철회 즉시 반영
- **승인 조건**:
  - [ ] 동의 변경 반영 SLA 충족
  - [ ] 동기화 실패 시 알림 및 재처리 가능

## 11. ACMS 설계 시사점

- Agentic CMS는 "콘텐츠 생성"뿐 아니라 "경험 의사결정"까지 다뤄야 함
- 개인화 규칙엔 설명가능성(왜 이 콘텐츠가 노출되었는지) 기능 필요
- 실험/성과 측정 루프를 CMS 내부 워크플로우와 통합해야 실효성 확보
- 데이터 지연/누락 시 기본경험 제공하는 회복탄력성 설계가 필수
